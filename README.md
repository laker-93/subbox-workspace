# subbox-workspace

Cross-repo entry point for working on the **subbox** platform with Claude. This repo
contains no application code — only the project-wide docs that orient an agent before
it makes changes across the two sibling repositories:

- `../subbox-app` — Electron/React music-player client (fork of Feishin)
- `../pymix` — FastAPI ETL backend (Rekordbox/Serato ↔ Subsonic; per-user Docker stacks)

Both are siblings of this repo, one level up. Paths are kept relative so the layout
works regardless of where the repos are cloned.

## Docs

- [`CLAUDE.md`](CLAUDE.md) — start here: what subbox is, how the repos relate, read order.
- [`docs/repositories.md`](docs/repositories.md) — the two repos and where each repo's own docs live.
- [`docs/architecture.md`](docs/architecture.md) — the system end to end.
- [`docs/integration.md`](docs/integration.md) — the pymix HTTP seam between client and backend.
- [`docs/adding-a-feature.md`](docs/adding-a-feature.md) — cross-repo feature workflow.

Repo-specific detail lives in each repo's own docs and is linked, never duplicated, here.

## Regenerating these docs

All Claude docs across the three repos are generated by `doc_gen.py` in this repo:

```bash
python doc_gen.py            # write all docs across the three repos
python doc_gen.py --check    # show what would change, write nothing
```
